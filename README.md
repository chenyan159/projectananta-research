# Investment Research Site

Static read-only research dashboard for Project Ananta. The site shows company reports, industry reports, ranking runs, valuation snapshots, factor scores, and data-quality alerts.

Public entry points:

- `https://projectananta.com/` after GitHub Pages finishes certificate issuance and `Enforce HTTPS` is enabled.
- `http://projectananta.com/` may work first while the certificate is still pending.
- Fallback: `https://chenyan159.github.io/projectananta-research/`

The public site is served by GitHub Pages from the committed `docs/` folder. Cloudflare is used only as DNS/registrar for `projectananta.com`; Cloudflare Tunnel, Access, Workers, R2, and local `cloudflared` are not required.

## Repository Layout

- `src/`: React dashboard UI.
- `etl/build-data.mjs`: reads local research materials and writes JSON data into `public/data/`.
- `etl/sync-docs.mjs`: replaces `docs/` with the latest Vite build output.
- `public/data/`: generated JSON used by local Vite builds. Do not hand-edit.
- `public/CNAME`: source custom-domain file. Keep as `projectananta.com`.
- `docs/`: GitHub Pages publish bundle. Generated by `npm run pages:prepare`; commit this folder after every update.
- `docs/CNAME`: generated copy of `public/CNAME`.
- `dist/`: temporary Vite build output. Ignored by git.
- `node_modules/`: npm dependencies. Ignored by git.

Removed/legacy local Tunnel folders such as `cloudflared/`, `bin/`, `logs/`, and startup scripts should not be recreated unless the deployment model changes away from GitHub Pages.

## Source Data Outside This Folder

The website is not maintained only from `tools/site`. The ETL reads these upstream research folders under `D:\drive\Investment`:

- `基本面/公司调研/公司索引.md`
- `基本面/公司调研/<分类目录>/*.md`
- `基本面/行业调研/行业索引.md`
- `基本面/行业调研/<分类目录>/*.md`
- `基本面/分析报告/投资排序/*.csv`
- `基本面/特征量化/量化评分/Fxx_*_量化评分_YYYY-MM-DD.md`
- `基本面/日度资料/每日金融数据/每日金融数据_YYYY-MM-DD.md`

Ranking CSV names currently recognized by the ETL:

- `投资排序_YYYY-MM-DD.csv` -> 平衡
- `激进投资排序_YYYY-MM-DD.csv` -> 激进
- `非常激进投资排序_YYYY-MM-DD.csv` -> 非常激进
- `投资排序_公司调研交叉检查_YYYY-MM-DD.csv` -> 交叉检查

Company reports are matched by ticker in the file name. Industry reports are matched by `标准行业名称` or file name-derived standard name. Feature files are grouped by `Fxx` and the latest dated file for each factor is used.

## Build And Publish

Run from `D:\drive\Investment\tools\site`:

```powershell
npm install
npm run pages:prepare
git status --short
git add .
git commit -m "Update research dashboard"
git push
```

`npm run pages:prepare` performs three steps:

1. `npm run build:data`: regenerates `public/data/`.
2. `vite build`: builds the static app into `dist/`.
3. `node etl/sync-docs.mjs`: replaces `docs/` with `dist/` for GitHub Pages.

Do not edit generated JSON under `public/data/` or `docs/data/` by hand. Fix the upstream Markdown/CSV files, then rerun `npm run pages:prepare`.

## Local Checks

After a build, check the generated metadata:

```powershell
Get-Content public\data\meta.json
```

Expected high-level fields include:

- `companyCount`
- `industryCount`
- `featureCount`
- `latestRankingDate`
- `latestFinancialDate`
- `qualityAlertCount`

For a local preview:

```powershell
npm run preview
```

Open `http://127.0.0.1:8080/`.

For the public site after pushing:

```powershell
curl.exe -I http://projectananta.com/
curl.exe -sS http://projectananta.com/data/meta.json
curl.exe -I https://projectananta.com/
```

If GitHub Pages is still issuing a certificate, `http://projectananta.com/` may work before HTTPS becomes available. Enable GitHub Pages `Enforce HTTPS` after the certificate is issued.

## Public Data Boundary

This repository and GitHub Pages site are public. Anything included in company/industry Markdown reports, ranking CSVs, or generated JSON can be viewed on the internet. The ETL redacts absolute `D:\drive\Investment\` prefixes inside report markdown, but it still publishes report text and relative source paths.

Before pushing, avoid placing private notes, credentials, account data, or non-public files in upstream research folders that the ETL reads.

## Cost Boundary

The intended operating model is GitHub Pages free hosting from a public repository plus Cloudflare DNS/Registrar for the domain. Do not enable Cloudflare Tunnel, Access, Workers, R2, paid analytics, or other paid Cloudflare products for this site unless the deployment model is explicitly changed.
